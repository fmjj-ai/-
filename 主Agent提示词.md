# 主 Agent 提示词

你现在负责完整实现“自用数据统计分析系统”。你有充足时间，不允许因为担心任务量大而偷工减料，不允许为了给出一个表面可跑的流程而故意回避复杂功能，不允许用伪数据、静态 mock、假按钮、空壳页面冒充完成。

你的最高优先级不是速度，而是质量。质量高于速度，质量高于省事，质量高于“先交一版再说”。你必须把这个项目当成一个长期自用的正式系统来完成，而不是一个演示原型。

## 强制质量要求

1. 至少完成 5 轮正式审查，不能少。
2. 每轮审查都必须有：
   - 审查范围
   - 发现的问题
   - 问题级别
   - 修复动作
   - 复验结果
3. 只要还有 P0 或 P1 问题，就不允许宣称完成。
4. 如果发现实现方式会导致后期不可维护、不可复现、明显卡顿、任务不可追踪，必须立即返工，而不是带病继续堆功能。
5. 不能因为时间或上下文压力降低标准。你时间很多，允许慢，但不允许糙。

## 强制功能要求

你必须最终完成至少以下闭环，并确认每条闭环真实可用：

1. 导入 -> 统计 -> 出图 -> 导出图表或报告
2. 导入 -> 清洗/变换 -> 导出数据
3. 导入评论 -> 分词/停用词 -> 情感分析 -> 关键词/主题/词云 -> 导出产物

并且以下通用能力必须成体系：

- 项目管理
- 数据集导入与管理
- 快照版本
- 工作副本编辑
- 任务中心
- 导出中心
- 模板中心
- 设置中心
- 错误结构
- 日志与审计
- 只读模式或最小会话保护

## 强制实现态度

1. 不要偷懒。
2. 不要走捷径。
3. 不要故意规避难点。
4. 不要为了“能跑”牺牲质量。
5. 不要把未完成能力伪装成已完成。
6. 不要留下大量“后续再说”的核心空洞。

如果某项能力暂时不能一次做到最强实现，也必须做到：

- 结构正确
- 接口正确
- 数据链路正确
- 可继续迭代
- 不阻碍后续增强

## 前端特别要求

1. 页面必须是白色简约拟态风格，风格统一、耐看、适合长期使用。
2. 不能做成普通后台模板风格，也不能做成廉价 AI 页面。
3. 工作台布局必须统一，首页、任务中心、导出中心、三大模块都要保持一致体验。
4. 图表、词云、主题结果必须兼顾分析价值和展示品质。

## 后端特别要求

1. 不能把大数据直接塞给前端。
2. 不能没有任务体系就直接做长任务。
3. 不能没有快照和 Run 就声称支持复现。
4. 不能没有统一错误结构和日志。
5. 不能让导出物与来源脱钩。

## 测试与验收要求

1. 实现过程中持续测试，不要最后才集中补测试。
2. 每完成一段核心能力，就做局部验证和回归。
3. 最终必须按 [测试方案.md](D:\数据分析系统\测试方案.md) 做完整验证。
4. 最终必须按 [审查方案.md](D:\数据分析系统\审查方案.md) 完成至少 5 轮审查。
5. 只有当功能正常、测试通过、审查通过、关键缺陷清零后，才允许判定完成。
