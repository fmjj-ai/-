# 自用数据统计分析系统 — 需求说明（v2，按三大模块）

> 写作口吻：以“需求者/使用者”的视角描述“我希望系统具备什么页面与功能”。  
> 重要约束：**不删减你的原始 idea 内容**（可规范表达、补充细化、增加缺失的必需功能）。  
> 模块划分：**情感分析 / 数据统计分析 / 数据处理分析**。  

---

## 0. 我希望系统整体长什么样（全局页面与通用能力）

### 0.1 首页（总览页）
我希望有一个**总览主页**作为系统入口与状态中心。

**页面内容**
1. 三大模块入口卡片（可点击进入）  
   - 情感分析  
   - 数据统计分析  
   - 数据处理分析  
2. 最近使用  
   - 最近导入的数据集 / 最近打开的项目  
   - 最近生成的图表 / 报告 / 导出文件  
3. 任务状态一览（很重要）  
   - 正在运行的任务（导入、计算、训练、导出等）  
   - 成功/失败数量与最近失败原因摘要  
4. 快捷入口  
   - 数据导入  
   - 导出中心  
   - 模板中心  
   - 设置（停用词/色卡/API Key 等）

**验收标准**
- 从首页能在 1 次点击内进入任意模块；能看到最近任务状态（运行中/成功/失败）和最近导出物。

### 0.2 通用布局（所有模块页面都保持一致）
我希望每个模块页面都有一致的“工作台式布局”，减少学习成本。

**页面骨架**
- 顶部：当前项目/数据集名称、保存状态、导出按钮、任务状态入口
- 左侧：可伸缩互动栏（功能区，可折叠/可拖拽宽度）
- 中间：数据表格（实时展示/编辑）
- 右侧/下方：结果面板（图表/曲线/模型结果/可视化/日志）

**验收标准**
- 左侧互动栏可折叠/展开；模块切换后布局一致；任何长任务都有进度展示入口。

### 0.3 数据导入与管理（通用）
我希望任何模块都能直接使用同一套“数据导入与管理能力”。

**支持格式（必须）**
- Excel（.xlsx/.xls）
- CSV
- JSON

**导入方式**
- 拖拽上传 / 点击上传
- 导入进度可见，导入失败给出原因（例如编码、格式损坏、字段不一致）
- 支持多文件 Tab 标签页切换
- 内置示例数据集（方便测试）

**数据集管理（建议作为独立页面/弹窗）**
- 数据集列表：名称、来源、导入时间、行列数、最近使用时间
- 支持：重命名、复制为新数据集、删除（必须二次确认）

**验收标准**
- 导入后立即可在表格中预览并可编辑；多文件可用 Tab 切换；导入失败不会产生半成品数据集。

### 0.4 数据表格展示与编辑（通用）
我希望上传数据后能**实时展示**，并且可以直接在表格里修改。

**表格能力**
- 可编辑表格（单元格级别）
- 列类型自动推断：数值/文本/日期/分类（并允许手动修正）
- 排序、筛选、搜索
- 分页展示（大数据必须分页/虚拟滚动）
- 新增/删除 行列（删除需二次确认）
- 冻结表头

**验收标准**
- 任何数据集都能在表格里进行编辑、筛选、搜索与分页；删除列会提示对下游图表/模型/导出的影响。

### 0.5 图表系统（通用：2D+3D + 极致美观 + 导出）
我希望图表既能满足分析，又能满足“汇报展示”的美观需求。

#### 0.5.1 2D 图表（必须）
我希望系统对任意导入的数据，都能快速生成常见 2D 图表，并且图表配置可以保存复用。

**支持的 2D 图表类型**
- 柱状图、折线图、饼图、散点图、箱线图、热力图、雷达图、漏斗图、词云等

**交互**
- 图表字段映射：我希望能选择用于 X/Y/分组/聚合 的字段
- 过滤器：我希望图表支持绑定筛选条件（并与表格筛选联动）

**验收标准**
- 任意数据集选择任意图表类型后能生成图表；保存后下次可一键复用配置到同数据集/新数据集（至少复用“字段映射+样式主题”）。

#### 0.5.2 3D 交互与展示（必须强调：不一定需要 3 个变量）
我希望系统支持“为了展示更酷炫”的 3D 图表模式：

**核心要求**
- 3D 图表**不一定需要三个变量**：  
  - 单变量图表（例如柱状图）也能切到 3D 模式（Z 轴可固定为装饰性深度）  
  - 目的：用于汇报展示、好看、可截图
- 支持 360° 交互：旋转、缩放、高透视角度、拖拽
- 支持 3D 曲面、3D 点云分布（按图表类型提供）
- 支持 2D/3D 自由转换

**验收标准**
- 任意 2D 图表都能一键切到 3D 展示模式（至少覆盖柱/散点/曲面类）；在 3D 视图中可旋转缩放并导出截图。

#### 0.5.3 美观与主题（必须：可选色卡 + 自定义）
我希望整体视觉“耐看、可长期使用、适合汇报”，并提供主题能力。

**预置主题色卡（至少）**
- 低饱和马卡龙
- 柔和唯美（偏暖/偏冷各一套）
- 简约朴素（黑白灰、米白）
- 莫兰迪
- 黑金商务（汇报用）
- 高对比（强调重点）

**自定义能力**
- 我希望能自由选择颜色，并能保存为自定义主题
- 我希望可配置：字体（标题/正文）、背景色、网格线/轴线样式

**验收标准**
- 切换主题后所有图表与页面关键元素同步变化；自定义主题可保存并在下次默认生效。

#### 0.5.4 图表导出（必须：SVG/PNG + 高清 + 透明背景）
我希望“一键导出配置或结果图片”：

**导出格式**
- SVG（矢量）
- PNG（高清）

**导出选项**
- PNG 高清倍率可选
- 可选透明背景
- 2D/3D 都支持导出

**验收标准**
- 导出 PNG 具备高清效果；选择透明背景后 PNG 背景为透明；SVG 可无损放大。

### 0.6 全部步骤支持导出（通用）
我希望系统的每一步都能导出“结果”与“复现配置”。

**必须支持的导出类型**
1. 处理后的数据 → CSV / Excel / Parquet
2. 分析结论/统计报告 → PDF / Markdown（支持 AI 生成的解读文字，并可编辑）
3. 特征工程配置 → JSON / YAML（可复用）
4. 图表（2D/3D）→ SVG（矢量）/ PNG（高清、可透明背景）
5. 文本主题可视化 → 可保存的 HTML（例如 pyLDAvis 导出）

**导出中心（建议独立页面）**
- 我希望有“导出中心”统一管理所有导出产物：
  - 文件类型、来源模块、生成时间、生成参数摘要、下载/删除

**验收标准**
- 任意导出产物都能在导出中心找到并再次下载；删除需要确认且不会误删原始数据集。

### 0.7 极致防卡顿策略（核心优化：必须写进需求）
我希望系统在**百万级数据或复杂计算**下仍保持“如丝般顺滑”，因此我明确提出以下性能策略作为需求约束（不是建议）：

#### 0.7.1 颜色与渲染懒加载（Lazy Rendering）
1. **颜色懒加载（Color Callback）**
   - 我不希望后端把 10 万个颜色码发给前端
   - 我希望后端只返回 `palette: ['#A', '#B', '#C']`
   - 我希望前端通过 `itemStyle: { color: function(params) { return palette[...] } }` 动态计算颜色
   - 目标：降低内存占用
2. **视口渲染（IntersectionObserver）**
   - 页面底部的图表、统计卡片：在滚入视口前只渲染 Skeleton（骨架屏）
   - 滚到位置才真正渲染（例如才 setOption）
   - 切换 Tab/折叠面板时：隐藏的图表实例必须释放资源（例如 dispose），释放 GPU 内存（含 ECharts/GL）
3. **Canvas/SVG 双模式**
   - 交互时默认 Canvas 模式（渲染快、内存低）
   - 导出时临时创建 SVG 实例进行无损渲染，导出完成即销毁

#### 0.7.2 后端聚合与数据截断
1. **聚合优先**
   - 前端图表只展示聚合数据点
   - 例：绘制直方图时，由后端先做 bin 计算，只返回固定数量（例如 50 个桶），而不是返回 100 万条原始记录
2. **服务端分页/流式加载**
   - 表格永远只展示 50–100 行数据
   - 排序、搜索通过后端处理（例如 SQL / Pandas 语义的服务端处理能力）

#### 0.7.3 Web Worker 异步流与 SSE 进度
1. **Web Worker（前端重计算）**
   - 导出大数据 CSV/Excel，或本地执行简单降维可视化时：必须放入 Web Worker，不阻塞主线程
2. **SSE 进度推送（长任务）**
   - 模型训练或长运行任务：后端通过 SSE 推送进度（如 `{"progress": 0.45}`）
   - 前端展示动画进度条，并允许取消

**验收标准**
- 在百万级数据集下：表格分页、筛选、滚动不出现明显卡顿；任何长任务都有进度与可取消能力；隐藏图表资源可释放，不因多次切换导致显存/内存持续上涨。

### 0.8 任务中心（通用：所有耗时操作都走任务）
我希望导入、困惑度计算、LDA、聚类/训练、导出大文件等都进入任务中心统一管理。

**页面：任务中心**
- 任务列表：类型、来源模块、状态（排队/运行中/成功/失败/取消）、进度、开始/结束时间、耗时
- 任务详情：参数摘要、日志、错误原因、重试/取消

**验收标准**
- 长任务必然出现在任务中心；失败任务能看到失败原因并可重试；取消后任务停止且状态正确。

### 0.9 模板中心（通用：为了“以后不用每次重写”）
我希望系统能沉淀“模板”，把我常用的分析套路固化下来。

**模板类型**
- 清洗模板（缺失/重复/异常处理组合）
- 图表模板（字段映射+样式主题）
- 统计分析模板（常用统计指标集合）
- 建模模板（特征工程+模型对比配置）
- 导出模板（报告结构+导出格式）

**验收标准**
- 任意一次分析操作都能“保存为模板”；模板能应用到新数据集并复用流程结构。

### 0.10 设置中心（通用：停用词/色卡/API Key/默认策略）
我希望设置集中管理，并且设置项能对三大模块生效。

**设置项（至少）**
- 停用词：默认哈工大停用词表 + 自定义停用词表管理
- 图表主题：预置色卡 + 自定义主题
- 外部 API：例如 DeepSeek API Key；是否允许外发数据（开关）
- 默认导出参数：PNG 默认倍率/是否透明背景、报告默认格式等
- 性能策略：默认分页大小、默认聚合桶数（例如直方图桶数）等

**验收标准**
- 设置修改后对新任务立即生效；涉及外部 API 的开关能明确提示风险与费用可能性。

### 0.11 技术实现约束（仅作为约束，不展开实现）
我希望实现优先参考以下技术栈（作为项目约束，不要求在需求里写代码）：
- Vue 3 + Vite（前端框架）
- Ant Design Vue（UI 组件库）
- Apache ECharts 5（2D 图表引擎）
- echarts-gl（3D 图表引擎：3D柱形/散点/曲面，鼠标旋转缩放拖拽）
- echarts-wordcloud（词云扩展）
- Pinia（状态管理）
- Axios（前后端通信）
- SQLite（轻量存储：用于项目/配置/任务/导出记录）

### 0.12 项目 / 工作区（Project & Workspace）
我希望系统以“项目”为单位组织所有资源，避免不同分析互相污染，也方便我长期积累与复用。

**需求**
- 项目内资源应隔离管理：数据集、流程（Pipeline）、运行记录（Run）、任务（Task）、图表/看板、导出物、模板、设置项（例如停用词、自定义主题）。
- 我希望在任何页面都能看到“当前项目”，并能快速切换项目。

**验收标准**
- 支持新建/重命名/归档项目；切换项目后，只显示当前项目的资源。
- 项目总览至少展示：数据集数、流程数、最近任务、最近导出、占用空间（可估算）。

### 0.13 数据集版本与快照（Snapshot，用于复现与回滚）
我希望对导入数据支持“快照/版本”，避免我在表格编辑后导致历史分析不可复现。

**需求**
- 支持对当前数据集创建只读快照（Snapshot），并能在运行流程时绑定快照版本。

**规则**
- 快照至少记录：行数、列数、schema（列名/类型）、创建时间、来源文件名与唯一标识（哈希或等价标识）。
- 我在表格编辑数据时，系统必须提示：这是对“工作副本”的编辑；如需可复现，应创建新快照或新版本。

**验收标准**
- 后续流程运行基于选定快照；UI 明确显示“当前使用的快照版本”。
- 至少支持回退到最近一次快照（最小回滚能力）。

### 0.14 分析流程记录与回放（Pipeline & Run）
我希望把“导入→清洗→变换→建模→可视化→导出”的每一步记录成可回放流程，以便以后不再重复写/改代码。

**需求**
- 每一步必须可序列化保存参数，并生成运行记录（Run），便于对比不同参数与结果。
- 支持从任意步骤开始重跑（例如从清洗后开始），并生成新的 Run 记录。

**规则**
- 每一步至少记录：步骤类型、参数、输入数据快照版本、输出产物引用（数据/模型/图表/文件）。
- 任一 Run 必须可查看“参数摘要 + 输入快照版本 + 输出产物入口”。

**验收标准**
- 流程可导出为 JSON/YAML；导入后可重建步骤与参数结构（流程形状一致）。
- 任一 Run 可对比上一次 Run 的参数差异（至少展示变更字段列表）。

### 0.15 通知中心与危险操作说明规范
我希望长任务完成/失败有统一通知入口；危险操作必须二次确认并明确影响范围。

**需求**
- 通知中心展示：任务名、状态、耗时、结果入口/错误入口。
- 危险操作（删除数据集/删除导出物/覆盖导出/清空缓存等）必须二次确认，提示“要做什么/为什么/影响/回滚方式（如有）”。

**验收标准**
- 任一任务完成/失败都能在通知中心看到并一键跳转到任务详情或产物。

### 0.16 统一错误结构（Error Handling）
我希望系统错误提示“可理解、可定位、可行动”，而不是只在控制台报错。

**需求**
- 统一错误结构：错误码（error_code）、用户提示（message）、定位信息（debug_info 可选）、下一步建议（suggestion）。

**验收标准**
- 常见错误必须有明确提示与建议动作：文件格式不支持、编码问题、列缺失、日期解析失败、外部 API 鉴权失败/限流/超时、内存不足、权限不足等。
- 失败任务在任务中心可查看错误详情，并支持复制错误信息与重试。

### 0.17 日志与操作记录（Logs & Audit）
我希望能追溯“我做过什么、系统做了什么、为什么失败”。

**需求**
- 每个任务产生结构化运行日志（阶段、耗时、关键参数摘要、警告），支持查看与下载。
- 记录关键操作：导入/删除数据集、创建快照、应用模板、导出、调用外部 API 等（单人也记录）。

**验收标准**
- 任务详情页可查看并下载日志（.log/.txt）。
- 操作日志可按时间筛选与关键字搜索；保留时长至少可配置（默认 30 天）。
- 外部 API 调用日志不记录敏感密钥，仅记录请求量/耗时/成功失败与原因摘要。

### 0.18 导出物命名规范与生命周期（Export Lifecycle）
我希望导出物长期使用不会把磁盘撑爆，并且能清楚知道每个文件从哪来。

**规则**
- 默认文件名包含：项目名、数据集名/快照号、导出类型、时间戳。
- 支持设置导出物自动过期（如 7/30/90 天）与手动删除（二次确认）。

**验收标准**
- 导出中心可看到导出物大小、创建时间、来源；到期后可自动清理或提示清理。

### 0.19 权限与会话保护（单人最低标准）
即使自用，我也希望有基础访问控制与“只读模式”，避免误操作或被他人随手打开看到数据与密钥。

**需求**
- 至少支持：启动口令/本地登录（二选一即可）+ 会话超时自动锁定（可配置）。
- 只读模式：禁止编辑表格、禁止删除数据/导出物、禁止修改全局配置；UI 给出明确提示。

### 0.20 大数据资源保护与缓存复用
我希望系统对重计算有资源保护与降级策略，并支持结果复用，避免重复等待。

**需求**
- 对重计算任务提供内存/时间保护与降级策略：抽样、聚合、分块、限制并发。
- 相同“数据快照 + 参数”的步骤支持缓存命中（例如困惑度曲线、TF‑IDF、LDA、聚类、回归等）。

**验收标准**
- 支持可配置阈值：最大行数/最大列数/最大并发任务；超阈值时提示并推荐方案（抽样/聚合/降低主题数等）。
- 二次运行相同参数时，任务详情显示“命中缓存/复用结果”。
- 支持手动清理缓存（按项目/按时间），带二次确认。

## 1. 情感分析模块（我希望它怎么用、有哪些页面与功能）

### 1.1 页面清单
我希望情感分析模块至少包含这些页面/子页面：
1. 情感分析工作台（核心页）
2. 停用词管理（设置子页，或在设置中心里也可）
3. 语料库管理（高置信度语料沉淀与导出）
4. 主题与可视化产物管理（pyLDAvis HTML、词云等产物的查看/下载）

### 1.2 页面：情感分析工作台

#### 1.2.1 页面区域
- 数据区：上传评论表格数据（Excel/CSV/JSON），支持多文件 Tab、示例数据
- 表格区：实时展示与编辑（通用表格能力）
- 左侧互动栏（流程开关与参数区）
  1. 选择文本列（我要指定哪一列是评论文本）
  2. 分词（jieba）
  3. 停用词（默认哈工大 + 自定义）
  4. 情感分析方式选择（SnowNLP / DeepSeek API）
  5. 高置信度语料沉淀（阈值可配）
  6. TF‑IDF 关键词提取（可选）
  7. LDA 主题提取（必须先困惑度曲线，再选主题数）
  8. pyLDAvis 主题可视化（导出 HTML）
  9. 词云图生成与自定义（颜色/线条/轮廓图/字体）
- 结果区：
  - 情感分析结果列（回写到原数据后）
  - 关键词结果、困惑度曲线、主题列表、pyLDAvis 预览入口、词云预览
- 导出区：
  - 导出处理后数据/结果/报告/配置/HTML/词云

#### 1.2.2 我希望的核心流程（主流程）
1. 我上传包含评论文本的表格数据
2. 我选择“文本列”
3. 我点击“分词（jieba）”，并开启“去停用词”（默认哈工大停用词表，可追加自定义停用词）
4. 我选择情感分析方式：
   - 方案 A：SnowNLP
   - 方案 B：DeepSeek API（我可配置 API Key，且系统要提示风险/费用/外发）
5. 系统输出情感分析结果，并把结果追加到原数据后面：
   - 情感得分（score）
   - 正/负面判断（polarity，或正/中/负按设置）
   - 置信度（confidence，如可得；否则标记为不可用）
6. 我可选继续做：
   - TF‑IDF 关键词提取
   - LDA 主题提取（先困惑度曲线，后选主题数）
   - pyLDAvis 可视化并导出 HTML
   - 生成词云（高度自定义）
7. 我保存本次流程，并可导出所有产物

#### 1.2.3 功能点明细（输入/输出/规则/边界/验收）

##### SA-01 分词（jieba）
- 输入：数据集 + 文本列 + 分词触发
- 输出：分词 tokens（可预览抽样）
- 规则：
  - 分词结果必须可预览（至少抽样 20 行）
  - 分词参数（例如精确/全模式等）作为可选配置项（不写实现）
- 边界：
  - 文本为空/全空时提示“无可分析文本”
- 验收：
  - 分词后能看到 tokens 预览与总词数统计（可选）。

##### SA-02 去停用词（默认哈工大 + 自定义）
- 输入：tokens + 停用词配置  
  - 默认：哈工大停用词表（必须内置）  
  - 自定义：我可以额外输入/上传停用词表，叠加到默认之上
- 输出：去停用词后的 tokens
- 规则：
  - 默认停用词必须始终可用
  - 自定义停用词必须能启用/禁用，并记录版本（便于复现）
- 边界：
  - 自定义停用词表为空时不报错
- 验收：
  - 开关自定义停用词后，tokens 预览发生变化；且系统显示“命中停用词数量”。

##### SA-03 情感分析（SnowNLP / DeepSeek API）
- 输入：
  - 预处理后的文本（或原始文本）
  - 模型选择：SnowNLP 或 DeepSeek API
- 输出：
  - 每行情感得分、情感标签（正/负等）
  - 必要元信息：使用的模型类型/版本标识、参数摘要
- 规则：
  - 选择 DeepSeek API 时，必须提示：
    - 可能产生费用
    - 可能存在数据外发风险
    - 我可以选择“抽样分析/脱敏/禁止外发”（至少提供一种）
- 边界：
  - 外部 API 失败：必须进入任务中心显示错误原因（网络/鉴权/限流等），支持重试
- 验收：
  - 情感分析完成后，在原数据后追加列；并能按情感标签筛选、统计与出图。

##### SA-04 高置信度语料沉淀（基础语料库）
- 输入：情感分析结果 + “高置信度阈值”
- 输出：基础模型语料库（可导出）
- 规则：
  - 阈值必须可配置
  - 语料库必须记录来源数据集、生成时间、阈值与模型类型
- 验收：
  - 能导出语料库（CSV/JSON）；再次打开仍可查看语料库信息。

##### SA-05 TF‑IDF 关键词提取（可选）
- 输入：tokens + TopK（可配）
- 输出：关键词列表（含权重）
- 验收：
  - 关键词结果可保存、可导出，并能用于词云输入源。

##### SA-06 LDA 主题提取（可选，但流程必须：先困惑度曲线）
- 输入：tokens + 困惑度计算范围（主题数候选区间）
- 输出：
  1. 困惑度曲线（必须先展示给我）
  2. 我选择主题数 K 后的主题结果（主题词、文档主题分布等）
- 规则：
  - 必须“先算困惑度 → 展示曲线 → 再让我选主题数”
  - 主题数 K 必须可自由选择
- 边界：
  - 计算耗时：必须进任务中心、有进度、可取消
- 验收：
  - 我能看到困惑度曲线并选择 K；生成结果可保存与导出。

##### SA-07 pyLDAvis 可视化（导出 HTML）
- 输入：评论文本 + LDA 模型结果
- 输出：pyLDAvis 可视化页面（HTML 文件，可保存/可管理）
- 规则：
  - HTML 必须作为“产物 Artifact”进入导出中心/产物管理
- 验收：
  - 点击产物能打开预览；能下载 HTML。

##### SA-08 词云图（强自定义：颜色/线条/轮廓/字体）
- 输入：tokens / TF‑IDF 关键词 / LDA 主题词
- 输出：词云图（预览 + 导出）
- 自定义要求（必须）
  - 颜色：可自选颜色 + 可选主题色卡
  - 是否要线条（描边）
  - 轮廓识别/计算机视觉（尝试）：允许我让系统从图像识别轮廓（可选实现，但需求要保留）
  - 轮廓图 mask：我可以上传黑白轮廓图  
    - 我可以选择用白色区域填充字或黑色区域填充字
  - 字体：可自定义字体
- 验收：
  - 词云能按 mask 形状生成；颜色与字体配置生效；导出为高清 PNG（可透明背景）。

## 2. 数据统计分析模块（我希望它怎么用、有哪些页面与功能）

> 我对该模块的期望是：**简单、快速、把“统计比例/分布/相关性/回归”一键做出来**，并能出图与导出报告。

### 2.1 页面清单
1. 统计分析工作台（核心页）
2. 图表与看板页（可选：将多个图表组织成汇报看板）
3. 报告页（可选：自动生成可导出的统计报告）

### 2.2 页面：统计分析工作台

#### 2.2.1 页面区域
- 数据区：导入/切换数据集（通用能力）
- 表格区：实时展示与编辑（通用能力）
- 左侧互动栏（统计能力）
  1. 数据清洗入口（缺失/重复/异常等）
  2. 数据变换入口（计算列/GroupBy/标准化等）
  3. 自动概览
  4. 描述性统计
  5. 分类统计（频率/TopN）
  6. 相关性矩阵与热力图
  7. 回归分析（线性/多项式 + 拟合线）
  8. 图表生成（2D/3D）与主题
  9. 导出（数据/图表/报告/配置）
- 结果区：统计表、热力图、拟合线图、图表预览

#### 2.2.2 功能点明细（输入/输出/规则/边界/验收）

##### ST-01 数据清洗（必须覆盖你原始清单）
我希望统计模块内也能直接使用以下清洗能力（与通用清洗一致）：
- 缺失值处理：删除 / 填充（均值/中位数/众数/自定义）
- 重复值检测与去除
- 异常值检测（IQR / Z-score）
- 数据类型转换
- 列重命名
- 条件替换（查找替换）

验收：
- 我能选择列并应用清洗策略；系统提示影响行数/缺失数变化；清洗后可导出新数据集。

##### ST-02 数据变换（必须覆盖你原始清单）
- 新增计算列（支持表达式）
- 数据分组聚合（GroupBy）
- 数据标准化 / 归一化

验收：
- 新列/聚合结果能立即用于统计与图表；变换配置可保存为模板。

##### ST-03 自动数据概览（Must）
- 输出：数据类型、行列数、缺失率/重复率（至少行列数与类型分布）
- 验收：进入页面后能自动生成概览，并可导出到报告。

##### ST-04 描述性统计（Must）
- 数值列摘要：均值/中位数/标准差/分位数
- 分类列摘要：唯一值数/频率分布/TopN
- 验收：结果可导出；TopN 的 N 可配置。

##### ST-05 相关性矩阵 & 热力图（Must）
- 输出：相关性矩阵与热力图
- 规则：
  - 计算范围可选（选择哪些数值列参与）
  - 相关系数类型可选：Pearson / Spearman / Kendall（默认 Pearson）
  - 大数据策略：超过阈值时自动抽样（必须记录抽样比例与随机种子，用于复现）
- 边界：
  - 若缺失值很多：必须提示缺失处理策略（删除缺失行/按列填充/跳过该列），并将选择记录到本次 Run 参数中
- 验收：
  - 热力图可导出为 PNG/SVG；
  - 在大数据下能快速返回（依赖聚合/抽样策略），且任务详情可查到抽样与参数记录。

##### ST-06 回归分析（Must：线性/多项式 + 拟合线）
- 输入：因变量列、一个或多个自变量列、回归类型（线性/多项式）
- 输出：拟合曲线可视化与指标摘要
- 规则：
  - 多项式阶数可配置
  - 训练/验证划分可配置（默认 8:2），并记录随机种子（默认固定）
  - 指标集合可配置：回归至少包含 R²、MAE、MSE（默认全开）
- 边界：
  - 自变量包含分类列时：必须提示并提供编码策略（例如要求先在“数据处理分析模块”完成 One-hot 或指定编码方式）
  - 存在缺失值/异常值时：必须提示其对结果影响，并允许我选择“剔除/填充/保留但标注”
- 验收：
  - 我能看到拟合线与指标摘要；
  - 能导出图表与拟合摘要；
  - 在任务详情/Run 记录中能看到：使用的列、划分策略、随机种子与指标配置。

##### ST-07 图表（2D/3D）与导出（通用能力在本模块必须可用）
- 验收：统计结果能直接一键转为图表；2D/3D 可切换；支持高清/透明导出。

##### ST-08 报告导出（Must：Markdown；Should：PDF）
- 输出：Markdown/PDF（支持 AI 生成的解读文字，并可编辑）
- 验收：报告至少包含：数据概览、关键统计表、主要图表、结论摘要、参数与筛选条件摘要。

## 3. 数据处理分析模块（我希望它怎么用、有哪些页面与功能）

> 我对该模块的期望是：把“特征工程 + 聚类/分类/回归 + 可视化对比 + 可复现配置导出”做成工作台。

### 3.1 页面清单
1. 数据处理/建模工作台（核心页）
2. 聚类实验页（可选：专门展示 K 寻优曲线、聚类结果与可视化）
3. 分类/回归实验页（可选：多模型对比）
4. 模板与配置复用页（可选：将特征工程/训练配置保存复用）

### 3.2 页面：数据处理/建模工作台

#### 3.2.1 页面区域
- 数据区：导入/切换数据集（通用能力）
- 表格区：实时展示与编辑（通用能力）
- 左侧互动栏（处理与建模管线）
  1. 基础处理：转编码、类型转换、清洗、变换
  2. 编码：独热编码、多热编码
  3. 聚类：K-Means / DBSCAN / HDBSCAN / MeanShift
  4. 分类与回归：RandomForest / XGBoost / LightGBM / Linear Models
  5. 深度学习：轻量级 MLP（仅用于对比）
  6. 可视化与导出
- 结果区：训练/聚类结果、评估摘要、可视化、日志
- 任务区：长任务进度（SSE）、取消/重试

#### 3.2.2 功能点明细（输入/输出/规则/边界/验收）

##### PR-01 转编码（必须保留）
- 输入：选择列 + 目标编码/转换策略
- 输出：转换后的列/新列
- 规则：转换策略必须可配置并可复现保存
- 验收：转换后表格可预览；导出数据无乱码（Unicode 保持完整）。

##### PR-02 独热编码（One-hot）
- 输入：选择分类列 + 编码参数
- 输出：独热特征列（列数扩展）
- 规则：是否保留原列可配置
- 验收：编码后可导出；并可直接用于聚类/分类/回归训练。

##### PR-03 多热编码（Multi-hot）
- 输入：选择多标签列 + 分隔符/解析规则
- 输出：多热特征列（或向量表示）
- 规则：分隔符等解析规则必须可配置并保存
- 验收：多热编码结果可预览抽样；导出后可复用。

##### PR-04 聚类（Clustering：你给出的算法清单必须支持）
- 支持算法：
  - K-Means
  - DBSCAN
  - HDBSCAN
  - MeanShift
- 我希望的自动化能力（必须保留）
  - 自动寻找最佳 K 值：手肘法 + 轮廓系数（至少输出过程曲线/评分）
- 可配置项（最小集，必须记录到 Run/任务详情以便复现）
  - K-Means：K 候选范围、最大迭代数、随机种子
  - DBSCAN：eps、min_samples
  - HDBSCAN：min_cluster_size（或等价核心参数）
  - MeanShift：带宽（bandwidth，或等价核心参数）
  - 抽样策略：抽样比例/最大样本数、随机种子（必须记录）
- 输出：
  - 聚类标签写回原数据（新增列）
  - 聚类结果可视化展示（2D/3D 皆可）
- 边界：
  - 大数据时允许抽样，并必须提示抽样策略（例如抽样比例/随机种子）
- 验收：
  - 我能看到“最佳 K 建议 + 曲线/评分 + 标签列 + 可视化”；结果可导出。
  - 任务详情/Run 记录中必须可查到：输入快照版本、算法与参数、是否抽样及抽样种子、输出标签列名、可视化产物链接。

##### PR-05 分类与回归（传统模型）
- 支持模型：
  - RandomForest
  - XGBoost
  - LightGBM
  - Linear Models
- 输出：
  - 评估指标（分类/回归分别给出摘要；指标集合可配置）
  - 预测结果（可写回为新列或输出结果表）
  - 多模型对比表（很重要）
- 规则：
  - 必填输入：目标列（label/target）必须选择；特征列可自动推荐并允许我手动排除
  - 训练/验证切分策略可配置（默认 8:2），并记录随机种子（默认固定）；配置可保存为模板
  - 指标集合可配置：
    - 分类至少包含 Accuracy、F1、AUC（默认全开）
    - 回归至少包含 R²、MAE、MSE（默认全开）
  - 预测写回策略可配置：写回原表新列 / 输出独立结果表（二选一）
  - 防止数据泄漏（规则要求）：切分必须发生在任何“基于全量统计的拟合/学习”之前（作为需求约束写入，不展开实现）
- 边界：
  - 类别极不平衡时：必须提示，并提供最小处理策略选项（占位即可，如 class_weight/重采样）
- 验收：
  - 同一数据集可对比至少 2 个模型；导出对比表与配置文件。
  - 任务详情/Run 记录中可查到：输入快照版本、特征列清单、切分策略与随机种子、指标集合、预测写回策略与输出产物链接。

##### PR-06 深度学习：轻量级 MLP（仅用于对比）
- 目标：仅用于与传统模型对比，不追求最强
- 输出：与传统模型同维度的评估摘要，用于对比
- 验收：能在同一数据集下生成对比结果，并导出对比表。

##### PR-07 “后续你帮我补充…”（必须保留为需求占位）
- 我希望在上述基础上继续扩展更多“机器学习/深度学习”等处理能力（具体清单后续补充，但此占位必须保留）

## 4. 通用质量、权限、安全与一致性要求（三大模块共同遵守）

### 4.1 数据质量（建议必须做最小集）
我希望系统能提供最小可用的数据质量能力，用来避免“垃圾进、垃圾出”：
- 缺失/重复/异常统计（与清洗联动）
- 质量规则（可选）：我可以配置规则并在导入/处理后自动检查（例如：某列不可为空、范围约束）
- 质量告警（可选）：规则失败进入任务中心并提示

### 4.2 权限与安全（自用最低标准）
即便自用，我也希望有基本的安全边界：
- 外部 API（DeepSeek）调用前必须提示：费用与数据外发风险，并允许我关闭外发
- API Key 必须在设置中心管理，不应出现在导出的报告明文中

### 4.3 术语与输出字段一致性
我希望系统对关键字段命名一致：
- 情感分析输出列命名规则（可配置）：score / polarity / label / confidence（如可得）
