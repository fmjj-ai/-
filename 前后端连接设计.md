# 前后端连接设计

## 1. 文档目标

本文档定义前后端之间的接口协议、状态同步方式、SSE 事件规范、错误结构、前端 store 与后端服务的映射，以及三大模块的联调主流程。

## 2. 通信原则

- 同步接口：REST + JSON
- 长任务进度：SSE
- 文件上传：`multipart/form-data`
- 文件下载：二进制流或下载地址

统一响应：

```json
{
  "success": true,
  "data": {},
  "meta": {}
}
```

统一错误：

```json
{
  "success": false,
  "error": {
    "error_code": "TASK_NOT_FOUND",
    "message": "任务不存在或已被删除。",
    "debug_info": {
      "task_id": "task_001"
    },
    "suggestion": "请刷新任务列表后重试。"
  }
}
```

## 3. 鉴权与会话

最小设计：

- 登录后获得会话令牌
- 所有写操作带鉴权
- 只读模式由后端返回能力标识，前端据此禁用编辑

## 5. 核心接口设计

### 5.1 数据导入

`POST /api/v1/projects/{projectId}/datasets/import`

返回：

```json
{
  "success": true,
  "data": {
    "task_id": "task_import_001",
    "dataset_id": "dataset_001"
  }
}
```

### 5.2 表格查询

`GET /api/v1/datasets/{datasetId}/table?page=1&page_size=50&sort=name&order=asc&keyword=测试`

返回：

```json
{
  "success": true,
  "data": {
    "columns": [],
    "rows": [],
    "pagination": {
      "page": 1,
      "page_size": 50,
      "total": 1000000
    }
  }
}
```

### 5.3 创建快照

`POST /api/v1/datasets/{datasetId}/snapshots`

### 5.4 流程运行

`POST /api/v1/pipelines/run`

请求体建议：

```json
{
  "project_id": "project_001",
  "module": "sentiment",
  "snapshot_id": "snapshot_001",
  "pipeline_id": "pipeline_001",
  "steps": [
    {
      "type": "tokenize",
      "params": {
        "text_column": "comment"
      }
    }
  ]
}
```

## 8. SSE 事件规范

事件类型：

- `task_created`
- `task_progress`
- `task_log`
- `task_success`
- `task_failed`
- `task_cancelled`
- `artifact_created`
- `notification_created`

事件格式：

```json
{
  "event": "task_progress",
  "data": {
    "task_id": "task_001",
    "run_id": "run_001",
    "progress": 0.45,
    "stage": "lda_training",
    "message": "正在训练主题模型"
  }
}
```

## 9. 错误联动规则

### 后端要求

- 所有错误必须带 `error_code`
- 高价值任务错误必须附 `suggestion`
- 外部 API 错误要区分鉴权、限流、超时、未知失败

### 前端要求

- 表单校验错误显示在字段附近
- 任务运行错误显示在任务详情
- 同时提供复制技术详情入口

## 10. 联调顺序建议

- 第一阶段：项目、数据集、表格、快照、任务中心
- 第二阶段：图表、导出、模板、设置
- 第三阶段：三大业务模块
- 第四阶段：通知、日志、生命周期清理、只读模式

## 11. 契约测试要求

每类接口都要做最小契约测试：

- 成功结构
- 失败结构
- 缺参错误
- 权限错误
- 超大数据边界

## 12. 联调完成标准

- 前端不依赖猜测字段
- 后端不返回不稳定结构
- 长任务全链路可追踪
- 导出物可从页面直接跳转
- 错误、日志、Run、Task、Artifact 可互相定位
