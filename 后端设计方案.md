# 后端设计方案

## 1. 文档目标

本文档定义后端的服务边界、模块拆分、任务体系、存储策略、数据模型、错误结构与质量约束。目标是让后端成为整个系统的稳定底座，而不是仅提供几个临时接口。

## 2. 总体定位

后端不是简单的 CRUD 层，而是以下五个角色的组合：

- 元数据中心
- 数据处理引擎
- 任务调度中心
- 导出与产物中心
- 可复现与审计中心

## 3. 技术建议

### 3.1 核心技术栈

- Python 3.11+
- FastAPI
- Pydantic v2
- SQLAlchemy 或 SQLModel
- SQLite
- Polars
- Pandas 兼容层
- scikit-learn
- xgboost
- lightgbm
- jieba
- SnowNLP
- pyLDAvis

### 3.2 存储原则

- SQLite 只存元数据
- Parquet 存快照与工作副本
- 文件系统存产物与日志

## 4. 后端分层架构

- API 层
- 应用服务层
- 任务编排层
- 领域能力层
- 持久化层
- 文件与产物层

## 6. 核心服务设计

### 6.1 项目服务

- 新建项目
- 重命名项目
- 归档项目
- 读取项目总览
- 统计项目占用空间

### 6.2 数据集服务

- 导入文件
- 创建数据集记录
- 生成首个快照
- 管理工作副本
- 删除或复制数据集

### 6.3 快照服务

- 创建只读快照
- 获取快照 schema
- 绑定快照到 Run
- 支持回退到最近快照

### 6.4 表格服务

- 服务端分页
- 排序
- 搜索
- 筛选
- 单元格编辑
- 行列增删
- 列类型推断与修正

### 6.5 流程与 Run 服务

- 序列化流程步骤
- 创建与查询 Run
- 记录参数差异
- 支持从任意步骤重跑

### 6.6 任务服务

- 创建任务
- 更新状态
- 进度推送
- 取消
- 重试
- 日志挂载

建议状态机：

- `queued`
- `running`
- `success`
- `failed`
- `cancelled`

### 6.7 导出服务

- 图表导出
- 数据导出
- 报告导出
- 配置导出
- HTML 产物导出

## 8. 长任务与执行器设计

建议使用本地任务编排器 + 进程池：

- API 创建任务记录
- 调度器分派到进程池
- Worker 执行任务
- 执行中持续写任务进度与日志
- SSE 订阅进度表或内存事件总线

## 9. 数据模型建议

元数据表至少包括：

- `projects`
- `datasets`
- `dataset_snapshots`
- `pipelines`
- `pipeline_steps`
- `runs`
- `run_step_results`
- `tasks`
- `task_logs`
- `artifacts`
- `templates`
- `settings`
- `notifications`
- `audit_logs`

## 10. 文件与缓存策略

缓存 Key 建议由以下组合生成：

- 输入快照 ID
- 步骤类型
- 参数摘要哈希
- 版本号

可缓存对象：

- 分词结果
- 去停用词结果
- TF-IDF
- 困惑度曲线
- LDA 结果
- 聚类寻优结果
- 模型训练结果
- 相关性矩阵

## 11. 错误结构

```json
{
  "success": false,
  "error": {
    "error_code": "DATASET_SCHEMA_MISMATCH",
    "message": "导入失败，字段结构不一致。",
    "debug_info": {
      "expected_columns": ["a", "b"],
      "actual_columns": ["a", "c"]
    },
    "suggestion": "请确认文件字段后重试，或作为新数据集导入。"
  }
}
```

## 12. 安全与会话

- 本地登录或启动口令
- 会话超时
- 只读模式
- API Key 加密存储
- 默认关闭外部 API 外发

## 13. 性能策略

- 表格接口只返回当前页
- 图表接口返回聚合结果，不返回全量原始点
- 大数据场景默认抽样，并记录策略
- 模型对比任务支持串行或受限并行

## 14. 可观测性

- 结构化日志
- 任务级日志
- 审计日志
- 外部 API 摘要日志
- 任务耗时、失败率、缓存命中率等指标

## 15. 后端完成标准

后端子 agent 必须至少交付：

- 元数据模型
- 文件存储策略
- 通用错误结构
- 任务调度基础设施
- SSE 进度流
- 三大模块核心服务
- 导出服务
- 日志与审计
- 缓存策略

通过标准：

- 能支撑三大模块闭环
- 大数据场景不会因错误设计直接卡死前端
- 所有结果可追踪、可复现、可导出
- 任务与快照体系完整
