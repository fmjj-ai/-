# 自用数据统计分析系统 Spec

## Why
需要一个高质量的、面向单人长期使用的数据统计与分析系统，支持从数据导入、清洗、统计到情感分析、可视化的全链路闭环。系统强调“复现性、高可用性和可追溯性”，以支持复杂的长期数据任务，绝不能仅仅是可跑通的演示原型。

## What Changes
- 初始化项目结构（前端：Vue3+Vite+AntDesignVue，后端：Python+FastAPI+SQLite）
- 构建通用底座：项目管理、数据集导入与版本快照、统一任务中心与导出中心
- 实现三大业务模块：
  - 数据处理分析模块
  - 数据统计分析模块
  - 情感分析模块
- 实现极致防卡顿策略：颜色懒加载、视口渲染、Web Worker 与 SSE 进度流
- 集成 ECharts (2D+3D) 进行结果可视化，并支持高质量导出 (SVG/PNG)

## Impact
- Affected specs: 完整的数据导入、处理、分析、可视化和导出能力。
- Affected code: 全新构建前后端系统架构，建立统一的数据管道和任务调度机制。

## ADDED Requirements
### Requirement: 统一底座能力
系统 SHALL 提供项目管理、数据集快照、导出中心、任务中心等通用能力，并且统一异常处理和日志审计。
#### Scenario: 导入与处理
- **WHEN** 用户导入超大 CSV 文件
- **THEN** 系统通过异步任务处理，前端通过 SSE 展示进度，完成后生成快照并在表格中分页展示。

### Requirement: 三大分析模块
系统 SHALL 提供数据处理、统计分析和情感分析能力。
#### Scenario: 情感分析闭环
- **WHEN** 用户选择文本列并启动情感分析
- **THEN** 后端通过 jieba 分词和 SnowNLP 情感打分等工具，生成词云与主题结果，并支持导出产物。
