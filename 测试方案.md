# 测试方案

## 1. 文档目标

本文档定义本项目的测试策略、测试分层、核心用例、测试数据、性能测试、回归测试与发布门槛。测试必须服务于“质量优先”，不能沦为只验证页面能打开。

## 2. 测试原则

- 测试必须覆盖功能、正确性、性能、稳定性、安全与可复现性
- 测试必须围绕需求闭环设计
- 测试结果必须可复查
- 核心链路必须自动化
- 大数据和长任务场景必须专项验证

## 3. 测试分层

### 3.1 单元测试

覆盖对象：

- 数据转换函数
- 指标计算函数
- 参数校验
- 错误结构包装
- 任务状态机
- 缓存 Key 生成

### 3.2 集成测试

覆盖对象：

- 导入到快照链路
- 表格接口与存储层
- 任务创建与取消
- 导出与产物索引
- SSE 进度推送
- 三大业务模块服务编排

### 3.3 端到端测试

覆盖对象：

- 用户级完整闭环
- 页面联动
- 任务中心与导出中心联动

### 3.4 性能与稳定性测试

覆盖对象：

- 百万级数据分页
- 大图表渲染
- LDA 长任务
- 聚类和模型对比任务
- 并发导出

## 4. 测试数据设计

至少准备以下数据集：

- 小型统计样例数据
- 大型百万级表格数据
- 评论文本数据
- 存在编码问题的异常文件
- 列缺失或 schema 不一致文件
- 含缺失值、重复值、异常值的数据
- 多标签分类列数据

## 5. 核心功能测试清单

### 5.1 通用能力

- 导入 Excel / CSV / JSON
- 导入失败回滚
- 多数据集 Tab 切换
- 表格分页、筛选、排序、搜索
- 行列增删二次确认
- 快照创建与回退
- 模板保存与复用
- 导出中心再次下载
- 任务中心重试与取消

### 5.2 情感分析模块

- 文本列选择
- jieba 分词
- 停用词启用与禁用
- SnowNLP 情感分析
- DeepSeek 风险提示
- 高置信度语料沉淀
- TF-IDF 关键词
- 困惑度曲线生成
- LDA 主题生成
- pyLDAvis HTML 预览与下载
- 词云 PNG 导出

### 5.3 数据统计分析模块

- 自动概览
- 描述性统计
- 分类统计
- 相关性矩阵
- 热力图导出
- 线性回归
- 多项式回归
- 报告导出

### 5.4 数据处理分析模块

- 类型转换
- 独热编码
- 多热编码
- K-Means
- DBSCAN
- HDBSCAN
- MeanShift
- 最佳 K 建议
- 分类/回归多模型对比
- 预测写回
- 结果导出

## 6. 长任务测试

以下任务必须专项验证：

- 大文件导入
- 困惑度计算
- LDA
- 聚类寻优
- 模型训练
- 大文件导出

每项都要验证：

- 创建任务
- 进度推送
- 取消
- 失败后查看日志
- 重试

## 7. 性能测试

### 7.1 前端性能指标

- 首页首屏可交互时间
- 工作台切换耗时
- 表格分页切换耗时
- 图表切换 2D/3D 耗时

### 7.2 后端性能指标

- 分页接口响应时间
- 聚合图表接口响应时间
- 导入解析耗时
- 统计任务耗时
- 模型训练任务耗时

### 7.3 资源指标

- 内存占用
- GPU/图表实例占用
- 任务取消后的资源回收

## 8. 安全与边界测试

- 未登录访问写接口
- 只读模式下的编辑操作
- API Key 是否暴露
- 删除导出物是否误删源数据
- 外部 API 失败后的错误结构
- 超大文件或异常格式导入

## 9. 回归测试

每次重大改动后至少回归以下主链路：

- 首页 -> 模块入口 -> 任务 -> 导出
- 数据集导入 -> 表格编辑 -> 快照 -> Run
- 三大模块各自核心闭环

## 10. 自动化建议

### 前端

- 组件测试
- 页面 E2E 测试
- 关键交互截图回归

### 后端

- API 测试
- 服务层测试
- 任务流程测试
- 导出产物校验

## 11. 缺陷管理

每个缺陷至少记录：

- 编号
- 所属模块
- 触发条件
- 实际结果
- 期望结果
- 严重级别
- 修复状态

## 12. 发布门槛

只有同时满足以下条件才允许通过：

- 单元测试通过
- 核心集成测试通过
- 三条主闭环 E2E 通过
- 长任务测试通过
- 性能基线达标
- P0、P1 缺陷为 0

## 13. 测试输出物

- 测试用例清单
- 自动化测试结果
- 性能测试结果
- 回归测试结果
- 缺陷清单与剩余风险
